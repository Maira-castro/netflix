01 — Apresentação

O que é
a Netflix?

Fundada em 1997 por Reed Hastings e Marc Randolph, a Netflix é uma plataforma de streaming de vídeo sob demanda (SVOD) que opera em mais de 190 países. Permite ao usuário assistir filmes, séries, documentários e conteúdos originais a qualquer hora, em qualquer dispositivo com conexão à internet, mediante assinatura mensal.

03 — Funcionamento

Como o
Sistema Opera

Da pressão de "play" pelo usuário até os pixels exibidos na tela, o sistema da Netflix executa uma complexa cadeia de operações distribuídas, envolvendo múltiplos processadores, memória, rede e armazenamento massivo.

Entrada (Input)
Clique no título, busca por texto, scroll no catálogo — capturados via teclado/mouse/touch e enviados via HTTPS

01

Interação do usuário → API Gateway

O usuário pressiona play. O cliente (navegador ou app) envia uma requisição HTTPS ao API Gateway da Netflix (Zuul), que autentica o token JWT do usuário e roteia a chamada para o microsserviço correto.

02

Microsserviço de Playback

O serviço de playback consulta o banco de dados de licenças (DRM), verifica o plano do assinante, seleciona o servidor CDN Open Connect mais próximo (baseado em geolocalização por IP) e retorna uma lista de URLs de segmentos de vídeo.

03

Streaming Adaptativo (DASH/HLS)

O cliente analisa a largura de banda disponível em tempo real. Um algoritmo ABR (Adaptive Bitrate) seleciona automaticamente entre múltiplas qualidades (360p, 720p, 1080p, 4K). O vídeo é dividido em segmentos de ~4 segundos, baixados sequencialmente e colocados em buffer na RAM.

// Exemplo simplificado de seleção de qualidade ABR function selecionarQualidade(bandwidth) { if (bandwidth > 15_000) return "4K_UHD"; if (bandwidth > 5_000) return "1080p_FHD"; if (bandwidth > 3_000) return "720p_HD"; return "360p_SD"; // fallback }
04

Decodificação pelo processador

Os segmentos chegam comprimidos no codec AV1 ou H.265. O processador (CPU) ou, preferencialmente, a GPU do dispositivo realiza a decodificação por hardware, convertendo dados binários comprimidos em frames de imagem. Isso economiza energia e permite reprodução em 4K em dispositivos móveis.

05

Telemetria e recomendação

Simultaneamente, o cliente envia dados de comportamento ao servidor (quanto assistiu, pausou, rebobinou). Esses dados alimentam o sistema de recomendação baseado em machine learning, que usa Apache Spark e TensorFlow para ajustar o catálogo personalizado do usuário.

04 — Representação e Tipos de Dados

Como os
Dados São
Tratados

A Netflix lida com tipos de dados radicalmente diferentes — de bytes de vídeo comprimido a registros JSON de perfil de usuário — cada um com formato e motivação técnica específica.

Vídeo
Binário / AV1
Segmentos de vídeo comprimidos
Arquivos binários codificados com AV1 ou H.265. A compressão reduz o tamanho em ~50% frente ao H.264 sem perda perceptível de qualidade, economizando largura de banda e custos de CDN.
Metadados
JSON / UTF-8
Catálogo e informações de títulos
Título, sinopse, gênero, elenco, duração, miniatura. Armazenados em JSON no ElasticSearch para buscas rápidas de texto completo. UTF-8 garante suporte a 190+ idiomas.
Perfil de usuário
Relacional / SQL
Dados de conta e preferências
Nome, e-mail (string), senha (hash bcrypt), plano (inteiro/enum), histórico de assistir (array de IDs). Armazenados no MySQL/CockroachDB com transações ACID para garantir consistência.
Comportamento
Event Stream / Avro
Telemetria e eventos de reprodução
Eventos como play, pause, seek, bitrate_change são serializados em Apache Avro (binário compacto) e transmitidos via Kafka para pipelines de análise em tempo real.
Imagens
WebP / JPEG
Miniaturas personalizadas (artwork)
A Netflix usa IA para gerar e selecionar qual thumbnail exibir para cada usuário. As imagens são servidas em WebP (30% menor que JPEG) via CDN com múltiplas resoluções (srcset).
Modelo de IA
Float32 / Tensor
Vetores de recomendação
Cada usuário e título é representado como um vetor de ponto flutuante em alta dimensão. A similaridade cossenoidal entre vetores determina o score de recomendação — processado em GPUs com TensorFlow.
05 — Requisitos Computacionais

Hardware e
Desempenho

O desempenho da Netflix é diretamente impactado pelo hardware do dispositivo do usuário. CPU, GPU, RAM e conexão de rede determinam a qualidade máxima de reprodução possível.

CPU
CPUs mais rápidas reduzem o tempo de decodificação de vídeo, diminuem latência no carregamento da interface React e aceleram a renderização de legendas e gráficos. Em dispositivos com CPUs fracas, a Netflix recorre ao H.264 (mais simples de decodificar) em vez de AV1.
GPU
GPUs com suporte a decodificação por hardware (NVDEC, AMD VCE, Apple Media Engine) liberam a CPU para outras tarefas, reduzem consumo de bateria em 40-60% e permitem reprodução estável de 4K/HDR sem travamentos.
RAM
A RAM armazena o buffer de vídeo (tipicamente 30 segundos à frente) e o estado da interface. Com pouca RAM, o SO força descartes do buffer, causando rebufferizações visíveis ao usuário.
Rede / E/S
A velocidade de rede é o principal gargalo de qualidade. A Netflix monitora o throughput real a cada 500ms e ajusta a qualidade dinamicamente. Redes instáveis (alta jitter) causam mais downgrade de qualidade do que velocidades baixas e estáveis.
